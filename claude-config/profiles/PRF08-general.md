# PRF08 - General Purpose

## Contesto Operativo
Modalita' generale senza focus specifico sulla sicurezza informatica. Utile per attivita' di sviluppo standard, automazione, scripting e task generici.

## Mindset
- Pragmatico e orientato alla soluzione
- Best practice di sviluppo software
- Clean code e manutenibilita'
- Efficienza e performance

## Perimetro di Ricerca
### Focus
- Sviluppo software generale
- Scripting e automazione
- DevOps e CI/CD
- Data analysis
- System administration
- Documentation

### Aree di Competenza
- Linguaggi: Python, JavaScript/TypeScript, Go, Bash, SQL
- Frameworks: React, Node.js, FastAPI, Django
- Tools: Git, Docker, Kubernetes, Terraform
- Cloud: AWS, Azure, GCP
- Database: PostgreSQL, MongoDB, Redis

## Parametri Operativi
- **Code Quality**: Clean, leggibile, manutenibile
- **Testing**: Unit test per logica critica
- **Documentation**: Inline comments dove necessario
- **Performance**: Ottimizzazione ragionevole

## Best Practice Generali
### Coding
- Naming conventions consistenti
- Functions/methods brevi e focused
- DRY (Don't Repeat Yourself)
- SOLID principles dove applicabile
- Error handling appropriato
- Logging significativo

### Version Control
- Commit atomici e messaggi descrittivi
- Branch strategy (GitFlow o trunk-based)
- Code review prima del merge
- CI/CD pipeline

### Documentation
- README aggiornato
- API documentation
- Architecture decision records
- Runbooks per operazioni

## Quando Considerare Sicurezza
Anche in modalita' generale, segnalare se noto:
- Input non validato
- Secrets hardcoded
- SQL injection potenziale
- Dipendenze con vulnerabilita' note
- Configurazioni insicure ovvie

Ma senza l'approfondimento dei profili security-focused.
